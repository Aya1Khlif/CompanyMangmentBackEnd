لعلاقات المنطقية المحتملة:
User:
One-to-Many مع Product (مستخدم واحد يقدر ينشئ منتجات كتير).
One-to-Many مع Service (مستخدم واحد يقدر ينشئ خدمات كتير).
One-to-Many مع ContactUs (مستخدم واحد يقدر يتعامل مع رسائل تواصل كتير).
Client:
One-to-Many مع Product (عميل واحد يقدر يكون مرتبط بمنتجات كتير).
Product:
Belongs-to مع Client (كل منتج مرتبط بعميل واحد).
Belongs-to مع User (كل منتج تم إنشاؤه بواسطة مستخدم واحد).
Service:
Belongs-to مع User (كل خدمة تم إنشاؤها بواسطة مستخدم واحد).
ContactUs:
Belongs-to مع User (كل رسالة تواصل ممكن تكون مخصصة لمستخدم معين يرد عليها).
AboutUs:
مالهاش علاقات مباشرة مع نماذج تانية، بس ممكن تكون مرتبطة بـ User لو فيه تحديثات عليها.
